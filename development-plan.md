## Technology Decisions

| Concern | Choice | Rationale |
|---------|--------|-----------|
| Primary language | Python 3.12+ | ML-heavy domain (demand forecasting, probabilistic models); first-class support for NumPy/SciPy/scikit-learn/LightGBM; stockpyl reference library is Python; strong async support via FastAPI |
| API framework | FastAPI | Auto-generates OpenAPI 3.1 spec (standards.md requirement); async request handling for concurrent forecast jobs; Pydantic model validation for request/response schemas and JSON Schema compliance |
| Database | PostgreSQL 16 + TimescaleDB | Relational backbone for reference data with JSONB flexibility; TimescaleDB hypertables for high-volume time-series (sales, demand signals, inventory snapshots, forecasts); continuous aggregates for dashboard KPIs; compression and retention policies for data lifecycle. Aligns with Data Model Suggestion 4 (time-series + analytics-first) as primary, with Suggestion 3's JSONB patterns for reference entities |
| Task queue | Celery + Redis | Async execution of forecast model training, demand signal ingestion, replenishment recommendation generation, and scheduled KPI computation. Redis doubles as cache for hot stock-level lookups |
| Event streaming | Apache Kafka (optional, Phase 10) | CloudEvents-compatible event bus for real-time demand sensing, EPCIS 2.0 event ingestion, and MCP server exposure. Deferred to later phase; Celery handles async in MVP |
| Frontend | Next.js 15 (React 19) + Tailwind CSS + shadcn/ui | Dashboard-heavy product (KPI views, forecast charts, replenishment recommendations); server components for initial load performance; charting via Recharts |
| ORM / query builder | SQLAlchemy 2.0 + Alembic | Async support with asyncpg; Alembic for migration management; raw SQL for TimescaleDB-specific operations (hypertables, continuous aggregates) |
| ML / forecasting | scikit-learn, LightGBM, Prophet, stockpyl | stockpyl (MIT) for classical inventory algorithms (EOQ, newsvendor, MEIO SSM/GSM); LightGBM for gradient-boosted demand forecasting; Prophet for seasonal decomposition; scikit-learn for pipeline orchestration |
| LLM integration | Anthropic Claude API (via anthropic SDK) | Natural-language replenishment explanations; external signal interpretation from unstructured sources; scenario narration. Prompt caching for repeated SKU explanation patterns |
| Authentication | OAuth 2.0 + JWT (python-jose) | RFC 6749/6750 compliance per standards.md; API key support for machine-to-machine integrations |
| Containerisation | Docker + docker-compose | Multi-service deployment (API, worker, frontend, PostgreSQL, Redis); consistent dev/prod environments |
| Testing | pytest + pytest-asyncio + httpx | pytest for unit/integration; httpx.AsyncClient for API testing; factory_boy for test data; testcontainers-python for PostgreSQL integration tests |
| Code quality | Ruff (lint + format), mypy (type check) | Ruff replaces black+isort+flake8; mypy for static type safety across data structures |
| Package manager | uv | Fast dependency resolution; lockfile support; replaces pip + pip-tools |
| API documentation | OpenAPI 3.1 (auto-generated by FastAPI) | Standards.md requirement; enables SDK generation and contract testing |
| Error responses | RFC 7807 Problem Details | Structured error payloads per standards.md; consistent across all API endpoints |

```python
from sqlalchemy.ext.asyncio import AsyncSession, create_async_engine, async_sessionmaker
from sqlalchemy.orm import DeclarativeBase
from invopt.config import Settings

class Base(DeclarativeBase):
    pass

async def create_engine(settings: Settings):
    engine = create_async_engine(
        settings.database_url,
        pool_size=settings.database_pool_size,
        max_overflow=settings.database_max_overflow,
        echo=settings.debug,
    )
    return engine

def create_session_factory(engine) -> async_sessionmaker[AsyncSession]:
    return async_sessionmaker(engine, expire_on_commit=False)
```

```python

    __tablename__ = "product"
    
    id: Mapped[uuid.UUID] = mapped_column(UUID, primary_key=True, default=uuid.uuid4)
    tenant_id: Mapped[uuid.UUID] = mapped_column(UUID, ForeignKey("tenant.id"), nullable=False)

    uom: Mapped[str] = mapped_column(String(20), default="EA")

    is_active: Mapped[bool] = mapped_column(Boolean, default=True)

    classification: Mapped[dict] = mapped_column(JSONB, default=dict)
    # {"abc": "A", "xyz": "X", "annual_revenue": 245000.00, "demand_cv": 0.12}

    attributes: Mapped[dict] = mapped_column(JSONB, default=dict)
    
    __table_args__ = (
```

```python
# src/invopt/schemas/common.py
from pydantic import BaseModel, Field
from uuid import UUID
from datetime import datetime

class PaginationParams(BaseModel):
    page: int = Field(default=1, ge=1)
    page_size: int = Field(default=50, ge=1, le=500)

class PaginatedResponse[T](BaseModel):
    items: list[T]
    total: int
    page: int
    page_size: int
    pages: int

class ProblemDetail(BaseModel):
    """RFC 7807 Problem Details"""
    type: str = "about:blank"
    title: str
    status: int
    detail: str
    instance: str | None = None

class TimestampMixin(BaseModel):
    created_at: datetime
    updated_at: datetime
```

**What**: Implement the InventoryService that processes inventory movements atomically: records the movement in the time-series table and updates the stock_level current-state row within a single transaction.

**Design**:

```python
# src/invopt/services/inventory_service.py
from enum import StrEnum

class MovementType(StrEnum):
    RECEIPT = "receipt"
    SHIPMENT = "shipment"
    ADJUSTMENT = "adjustment"
    TRANSFER_IN = "transfer_in"
    TRANSFER_OUT = "transfer_out"
    RETURN = "return"
    DAMAGE = "damage"
    CYCLE_COUNT = "cycle_count"
    SALE = "sale"

class InventoryService:
    def __init__(self, session: AsyncSession):
        self.session = session
    
    async def record_movement(
        self,
        tenant_id: UUID,
        product_id: UUID,
        location_id: UUID,
        movement_type: MovementType,
        quantity: Decimal,
        reference_type: str | None = None,
        reference_id: UUID | None = None,
        context: dict | None = None,
    ) -> StockLevel:
        """
        Atomically record a movement and update stock level.
        Positive quantity = stock increase (receipt, return, transfer_in)
        Negative quantity = stock decrease (shipment, sale, damage, transfer_out)
        """
        async with self.session.begin_nested():
            # Upsert stock_level with SELECT FOR UPDATE
            stock = await self._get_or_create_stock_level(tenant_id, product_id, location_id)
            
            # Apply movement to stock position
            match movement_type:
                case MovementType.RECEIPT | MovementType.RETURN | MovementType.TRANSFER_IN:
                    stock.on_hand += abs(quantity)
                case MovementType.SHIPMENT | MovementType.SALE | MovementType.DAMAGE | MovementType.TRANSFER_OUT:
                    stock.on_hand -= abs(quantity)
                case MovementType.ADJUSTMENT | MovementType.CYCLE_COUNT:
                    stock.on_hand += quantity  # can be positive or negative
            
            # Insert time-series movement record
            await self.session.execute(
                text("""
                    INSERT INTO ts_inventory_movement (time, tenant_id, product_id, location_id, 
                        movement_type, quantity, reference_type, reference_id, context)
                    VALUES (:time, :tenant_id, :product_id, :location_id, 
                        :movement_type, :quantity, :reference_type, :reference_id, :context)
                """),
                {
                    "time": datetime.now(timezone.utc),
                    "tenant_id": tenant_id,
                    "product_id": product_id,
                    "location_id": location_id,
                    "movement_type": movement_type.value,
                    "quantity": quantity,
                    "reference_type": reference_type,
                    "reference_id": reference_id,
                    "context": json.dumps(context or {}),
                }
            )
            
            return stock
```
